FeedMe

back

FeedMe is a collaborative news aggregator. Users can read, aggregate and categorize their favorite feeds, as well as see (and comment on) what everyone's reading.

features:

FeedMe single page Backbone.js app that consumes a Rails JSON API.

Uses redis and resque background workers to keep feeds up to date without sacrificing load time. Takes advantage of unicorn's multiple processes to maximize utilization of limited resources on the Heroku platform.

Takes advantage of event aggregation to implement a keyboard interface and flash message system that allows for communication accorss Backbone views.

Custom built authentication system that leverages BCrypt for encrypting and uses ActiveMailer and Sendgrid for activation and password reset mailings.

ReActiveRecord

back

Uses metaprogramming to construct a lightweight Ruby implementation of an ActiveRecord-style ORM

features:

Reflects on database columns to generate accessors for the ORM's model.

Queries return relation objects that facilitate the lazy loading of results - minimizing actual calls to the database.

Implements model associations including: has_many, belongs_to, and has_one_through.

Implements Relation#include by storing the results of a query in a closure, and using #send and #define_method to build a method on the original model that accesses the closure.

chess

back

A full-featured command-line implementation of chess.

features:

Uses the colorize gem and unicode to display the board to the terminal.

Can import or export a game state into an easy-to-read hash, making it easy to set up any initial position.

Takes advantage of object oriented design patterns to DRY out the code for pieces and movement.

Recursively duplicates the board to calculate the legality of possible moves.
